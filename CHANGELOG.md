# 更新日志 (Changelog)

本文档记录 AI 小说助手 (AInovel-Master) 各版本的功能与改动。

---

## [v0.2.4] - 2025-03-17

### 卡片更新与联网查询

- **取消自动更新与续写后弹窗**：移除「自动更新」开关、续写完成后根据正文更新卡片的逻辑，以及「接受更新到正文」后弹出的卡片更新确认流程。
- **一键更新**：右侧栏下方新增「一键更新」按钮；点击后读取当前小说最新正文与简介，由 AI 重新提炼所有卡片的描述并直接写回，无需用户逐条确认。
- **单卡片更新**：每个卡片新增「更新」按钮；点击后根据最新小说内容仅重新提炼该卡片，弹窗展示修订前后对比，用户确认后再覆盖。
- **联网查询**：每个卡片新增「联网查询」按钮；根据卡片名称与描述（如有）调用 AI 联网搜索并提炼成卡片内容，直接更新该卡片。

---

## [v0.2.5] - 2026-03-18

### 章节大纲自动生成

- 左侧章节列表新增 **Outline**：录入全书梗概/主线/开头-前期-中期-后期走向、预计总章数、每章目标字数等，一键生成分章标题与每章梗概并批量创建章节。
- 新增接口 `POST /api/chapters/generate-from-brief`，并将全书大纲追加写入小说简介。

### AI 进度与可观测性

- 顶部新增全局 AI 进度条：显示任务名称、阶段（读取/思考/联网检索/生成中）、实时详情与“已等待Xs”。
- `POST /api/ai/generate` SSE 新增 `status` 事件，前端可实时展示“正在读取上下文/筛选卡片/请求模型/等待首 token”等状态。
- 部署初期：错误提示改为不自动消失，便于复制与排查（需手动关闭）。

### 卡片引入与交互优化

- 引入卡片弹窗支持默认折叠、列表区域独立滚动、全部展开/折叠，避免多条目时内容显示不全。
- 引入候选生成更稳健：改进 JSON 解析、空结果重试，并提供正文切分兜底候选，降低“无可引入条目”的概率。
- 卡片操作按钮（Sync/Web）提高对比度与可读性。

### 联网搜索扩展支持

- 除 xAI/Grok 外，支持在「设置」中声明模型/接口 **支持联网工具**（写入 `extra_config_json.supports_web_search`），从而在发送时启用“联网”开关（适配 GLM5 等工具型接口）。

## [v0.2.3] - 2025-03-17

### 右侧属性卡片（文本化）

- **角色 / 世界观 / 设定** 卡片编辑改为「单一详细描述」文本框，不再拆分姓名/性格/外貌等字段；引导提示用户输入更完整的描述内容。
- **AI 注入优化**：续写时按“关键词”选择相关卡片注入提示（默认关键词为卡片名称，可在卡片内容 JSON 中额外提供 `keywords` 字符串增强匹配）。
- **兼容旧数据**：旧版本多字段 `content_json` 仍可读取并合并为描述文本展示与注入。

---

## [v0.2.2] - 2025-03-17

### 界面与交互

- **弹窗毛玻璃**：降低遮罩不透明度与模糊强度，弹窗后方页面更清晰可见。
- **小说家位置**：小说家从独立页签改为左侧栏顶部列表，与小说列表、章节列表同屏展示；三块区域外框 + iOS 风格圆角与滚动条；加号按钮增加圆形毛玻璃轮廓。
- **进度条**：AI 状态由胶囊改为标题下方、页面正中的长进度条，渐变填充、每次随机颜色，动画流畅，显示「准备上下文 / AI 思考中 / 正在生成小说」等状态。

### 新建章节

- 移除「根据梗概生成」与「AI 自动命名」，新建章节默认名称为「第一章」「第二章」「第三章」… 按顺序递增，仍可手动修改。

### 编辑区布局

- 生成内容过长时，下方输入框固定于底部不再被顶跑；编辑区可滚动查看长文与 AI 输出。
- AI 输出区域与用户输入框增加边框与毛玻璃样式，与整体页面风格一致。

## [v0.2.1] - 2025-03-17

### 小说家模块

- 左侧栏新增「小说家」页签，可创建、编辑、删除不同类型的小说家（如东方玄幻小说家、言情小说家）。
- 小说家可配置：名称、编写风格、排版方式；AI 续写时将选中的小说家风格注入系统提示并据此输出。

### 新建小说弹窗

- 新建小说改为弹窗录入：小说名称、小说简介、世界观（可选）。
- 若填写世界观，创建小说后自动在右侧卡片栏生成「世界观」卡片。

### 新建章节弹窗

- 新建章节改为弹窗录入：章节名称、内容梗概、目标字数、是否 AI 自动生成章节名称。
- 支持「根据梗概生成」按钮，调用 AI 生成章节标题；开启「AI 自动命名」后创建时自动根据梗概生成名称。
- 章节支持梗概与目标字数字段，续写时自动拼入提示词。

### AI 续写确认

- 续写生成完毕后不再自动并入正文，改为展示「接受」「重新生成」按钮。
- 用户确认接受后再合并进正文并保存；可多次点击重新生成直至满意。

## [v0.2] - 2025-03-17

### 前端深度优化

- **iOS 质感重构**：大圆角、毛玻璃（backdrop-filter）、柔和阴影与弹性交互动画。
- **Logo**：专属 Logo 替换左上角纯文本标题。

- **顶部 iOS 胶囊状态条**：用户提交续写后，顶部展示流畅的胶囊进度条，显示「准备上下文 / AI 思考中 / 正在生成小说」等状态，与 SSE 流式响应联动。

## [v0.1] - 初始版本

### 基础框架

- Vue 3 + Vite + TypeScript 前端，FastAPI + SQLAlchemy (SQLite) 后端。
- 三栏式布局：左侧小说/章节列表、中间编辑区、右侧属性卡片栏。
- Windows 本机调试与 Linux Docker 部署支持。

### 小说与章节

- 小说列表与章节列表完整 CRUD，右键重命名/删除。
- 基于 Tiptap 的富文本章节编辑，防抖自动保存。

### AI 与设置

- 设置弹窗：自定义 API Key、服务商（OpenAI / Grok / Anthropic / 自定义）、模型、代理地址、Grok 联网搜索。
- SSE 流式续写，实时展示「AI 续写中」并自动合并进正文。

### 属性卡片

- 右侧角色/世界观/设定/剧情线等卡片 CRUD，结构化编辑与「自动更新」开关。
- AI 续写时自动注入卡片内容到系统提示；续写结束后可选自动分析正文并更新卡片。

### 体验优化

- 章节列表拖拽排序。
- 全局 API 错误提示（axios 拦截器）。
